# LINE → Gemini → メルカリ 自動下書き作成システム　セットアップガイド

## 全体の流れ

```
LINE に商品画像を送る
      ↓
サーバーが受け取り（Railway）
      ↓
Gemini Vision で商品名・価格・説明を自動生成
      ↓
Playwright でメルカリ出品フォームを自動入力
      ↓
下書き保存 → LINE で完了通知
```

---

## STEP 1 ｜ Gemini API キーを取得する（無料）

1. https://aistudio.google.com/app/apikey を開く
2. 「Create API key」をクリック
3. 表示された API キーをコピーしてメモしておく

---

## STEP 2 ｜ LINE 公式アカウントの設定を確認する

1. https://developers.line.biz/ を開く
2. 作成済みのチャネル → **Messaging API** タブを開く
3. 以下の2つをメモしておく：
   - **チャネルアクセストークン**（長期）
   - **チャネルシークレット**

---

## STEP 3 ｜ メルカリのクッキーをエクスポートする

1. **Chrome** で https://jp.mercari.com にログインする
2. `F12` → **Console** タブを開く
3. `export_cookies.js` の中身を全部コピーして Console に貼り付け → Enter
4. 「✅ クッキーをコピーしました！」と表示される
5. クリップボードの内容（JSON）をテキストエディタなどに一時保存しておく

> ⚠️ クッキーは数週間〜数ヶ月で失効します。メルカリにログインできなくなったら再実行してください。

---

## STEP 4 ｜ Railway にデプロイする

### 4-1. GitHub にコードをアップロード

1. GitHub で新しいリポジトリを作成（例：`mercari-line-bot`）
2. このフォルダの中身を全てアップロード

### 4-2. Railway でプロジェクトを作成

1. https://railway.app でアカウント作成（GitHub でログイン可）
2. 「New Project」→「Deploy from GitHub repo」
3. 作成したリポジトリを選択
4. Railway が自動で Dockerfile を検出してビルド開始

### 4-3. 環境変数を設定する

Railway のプロジェクト → **Variables** タブで以下を追加：

| 変数名 | 値 |
|--------|-----|
| `LINE_CHANNEL_ACCESS_TOKEN` | STEP 2 でメモしたトークン |
| `LINE_CHANNEL_SECRET` | STEP 2 でメモしたシークレット |
| `GEMINI_API_KEY` | STEP 1 でメモした API キー |
| `MERCARI_COOKIES` | STEP 3 でコピーした JSON 全体 |

### 4-4. デプロイ完了を確認

- Railway のダッシュボードでビルドログを確認
- 「Deploy succeeded」と表示されたら OK
- **Settings** → **Domains** でサーバーの URL をコピー（例：`https://xxx.railway.app`）

---

## STEP 5 ｜ LINE の Webhook URL を設定する

1. LINE Developers → チャネル → **Messaging API** タブ
2. **Webhook URL** に以下を入力：
   ```
   https://あなたのRailwayURL.railway.app/webhook
   ```
3. 「Verify」をクリック → 「Success」と表示されれば完了
4. **Webhook の利用** をオンにする

---

## 動作確認

1. LINE アプリでボットに商品の写真を送る
2. 「📸 画像を受け取りました！商品情報を解析中です...」と返信が来る
3. 30〜60秒後に「✅ メルカリ下書きを作成しました！」と通知が来る
4. メルカリアプリ or ブラウザで **下書き一覧** を確認する

---

## トラブルシューティング

### 「クッキーが失効しています」と通知が来た場合
→ STEP 3 を再実行して MERCARI_COOKIES を更新する

### 下書きが作成されない（商品情報は届く）場合
→ メルカリのUIが変更された可能性があります。  
　 Claudeに「メルカリの下書きを作成して」と伝えると手動で対応できます。

### LINE Webhook が「Verify」で失敗する場合
→ Railway のデプロイが完了しているか確認する  
→ URL の末尾が `/webhook` になっているか確認する

---

## クッキーの更新について

メルカリにログインできなくなったら（通常 1〜3 ヶ月ごと）：

1. Chrome で https://jp.mercari.com を開いてログイン
2. `export_cookies.js` を Console で再実行
3. Railway の `MERCARI_COOKIES` 変数を新しい値に更新
4. Railway が自動的に再デプロイされる
